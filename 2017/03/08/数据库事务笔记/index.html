数据库事务笔记

发表于 2017-03-08

基本概念

数据库事务（简称：事务）是数据库管理系统执行过程中的一个逻辑单位，由一个有限的数据库操作序列构成。简单来说，一个事务是可以被看作一个单元的一系列SQL语句的集合。

一个数据库事务通常包含了一个序列的对数据库的读/写操作。它的存在包含有以下两个目的：

为数据库操作序列提供了一个从失败中恢复到正常状态的方法，同时提供了数据库即使在异常状态下仍能保持一致性的方法。当多个应用程序在并发访问数据库时，可以在这些应用程序之间提供一个隔离方法，以防止彼此的操作互相干扰。当事务被提交给了DBMS（数据库管理系统），则DBMS（数据库管理系统）需要确保该事务中的所有操作都成功完成且其结果被永久保存在数据库中，如 果事务中有的操作没有成功完成，则事务中的所有操作都需要被回滚，回到 事务执行前的状态;同时，该事务对数据库或者其他事务的执行无影响，所有的事务都好像在独立的运行。

但在现实情况下，失败的风险很高。在一个数据库事务的执行过程中，有可能会遇上事务操作失败、数据库系统/操作系统失败，甚至是存储介质失败等情况。这便需要DBMS对一个执行失败的事务执行恢复操作，将其数据库状态恢复到一致状态（数据的一致性得到保证的状态）。为了实现将数据库 状态恢复到一致状态的功能，DBMS通常需要维护事务日志以追踪事务中所有影响数据库数据的操作

四大特性(ACID)

原子性(Atomicity)

指的是事务是一个不可分割的工作单位，事务的操作要么都发生，要么都不 发生

一致性(Consistency)

事务前后的数据完整性必须保持一致

隔离性(Isolation)

事务的隔离性是指多个用户并发访问数据库时，一个用户的事务不能被其他的用户事务所干扰，多个并发事务之间数据要相互隔离

多个线程开各自事务操作数据库中的数据时，数据加系统要负责隔离操作，以保证各个线程在获取数据时的准确性

3.如不考虑隔离性，可能会引发如下的问题:

脏读

指上事务读取了另外一个事务未提交的数据。这个是非常之危险的，假设 A向B转账100000元，对应的SQL语句是：

update account set money=money+100000 while name=’b’

update account set money=money-100000 while name=’a’

当第一条SQL执行完成后，第2条还没有执行(A没有提交时)，如果此时B查询自己的帐户，就会发现自己多了100000元钱。如果A等B走后再进行事务的回滚，这个时候B就会损失100000元！这个是因为这个B开一个事务去读了那个A事务还没有提交的数据所造成的！

不可重复读

指在一个事务内读取表中的某一行数据，多次读取结果不同。(一个事务读取到的另外一个事务提交的数据)

举一个简单的例子，就是中央人银行每个月的月底都会生成这一个月的一个财务报表，这个时候就开启一个事务生成了A报表和B报表分别送给这个总理和主席，但A报表和B报表之间有储存业务的产生，这个时候B报表生成的总额和A报表生成的总额是不一样的。就是因为这个事务读取了另外一个事务提交的数据了！这种情况有时候好，但有时候就出问题了！

虚读(幻读)

指在一个事务内读取到了别的事务插入的数据，导致前后不一致。这个和不可重复读不一样，不可重复读是指某一行的数据，这个是指整个表的数据，第一次读有100条数据，但第二次读就出现了200条数据，前后读取是不一致的问题出现了！这种情况和不可重复读影响都是一样的，有时候好，但有时候就出问题了。

数据库定义了四种的隔离级别

1）Serializable :可避免脏读、不可重复读、虚读的发生。(串行化，这个 是性能最差的，是单线程的操作!就是有一个事务在进行，那个事务还没有commit的时候，这个事务作任务操作都不能，都必需要等到那个事务commit时候才能进行！)

2）Repeatable read : 可避免脏读、不可重复读的情况发生。(可重复读)

3）Read committed : 可避免脏读情况发生。(读已提交)

4）Read uncommitted :最低的级别，以上情况均无法保证。(读未提交)

这四种隔离的级别越往上性能就越低！可根据需求调整！

在MySQL可以通过set transaction isolation level 设置事务隔离级别，select @@tx_isolation 查询当前事务隔离级别

持久性(Durability)

持久性是指一个事务一旦被提交，它对数据库中的数据的改变就是永久性的 ，接下来她即使数据库发生故障也不应该对其有任何的影响！
