设计模式学习笔记-责任链设计模式

发表于 2017-04-10

今天做项目的时候，需要用到Filter技术来过滤请求，很好奇里面是如何实现这个Filter链的调用的，查阅下资料，原来是责任链设计模式的一个应用。简单做下学习笔记。

责任链设计模式

责任链设计模式是设计模式中的十一个行为模式的其中一种，使用频率比较多，在一些框架的设计里面都会见到它的影子。职责链模式通过建立一条链来组织请求的处理者，请求将沿着链进行传递，请求发送者无须知道请求在何时、何处以及如何被处理，实现了请求发送者与处理者的解耦。

在软件开发中，如果遇到有多个对象可以处理同一请求时可以应用职责链模式，例如在Web应用开发中创建一个过滤器(Filter)链来对请求数据进行过滤，在工作流系统中实现公文的分级审批等等，使用职责链模式可以较好地解决此类问题。

例子一

在《设计模式之禅》这本书中的责任链设计模式说到一个中国古代的女性“三从四德”的例子。古代的女性们在未婚前从父，嫁后从夫，夫死后从子。假如一个妇女要去行街购物，没有结婚需要请求下父亲，结婚后就请示丈夫，如果结婚后丈夫去世了，如果有儿子就请求儿子了。父亲->丈夫->儿子这三个角色构成了一条处理链。

实现

1、定义一个女性的接口，getType返回这个女性是女儿还是妻子还是母亲。getRequest这个返回这个女性的请求。

public interface IWomen {
    int getType();

    String getRequest();
}

2、一个简单的IWomen的实现类:

public class Women implements IWomen {
    private int type;
    private String request;

    public Women(int type, String request) {
        this.type = type;
        this.request = request;
    }

    @Override
    public int getType() {
        return this.type;
    }

    @Override
    public String getRequest() {
        return this.request;
    }

}

3、链上的角色，属于处理者，他们都共同只有一个特点就是:处理IWomen的请求，如果该IWomen的类型是自己处理得到的就处理，处理不到的就交给下一个处理者。处理者都引用着下一个处理者，并且会定义自己的处理方法。那么处理者应该设计成抽象的，因为他们拥有一个共同的操作方法。然后根据各自的特点派生出各自的处理者。

public abstract class Handler {
    public final static int FATHER_LEVEL_REQUEST = 1;
    public final static int HUSBAND_LEVEL_REQUEST = 2;
    public final static int SON_LEVEL_REQUEST = 3;

    // 下一个条处理链
    private Handler next = null;
    // 能处理的级别
    private int leve = 0;

    public Handler(int leve) {
        this.leve = leve;
    }

    //处理方法
    public final void handleMessage(IWomen women) {
        if (women.getType() == this.leve) {
            response(women);
        } else if (next != null) {
            next.response(women);
        } else {
            System.out.println("没有处理者，默认同意");
        }

    }

    //设置下一个处理者
    public void setNext(Handler handler) {
        this.next = handler;
    }

    //各自的响应方法
    abstract void response(IWomen women);

}

父亲处理者:

public class Father extends Handler {

    public Father() {
        super(FATHER_LEVEL_REQUEST);
    }

    //父亲自己的处理方法
    @Override
    void response(IWomen women) {
        System.out.println("------女儿向父亲请求--------");
        System.out.println("请求 : " + women.getRequest());
        System.out.println("父亲回复:同意");
    }

}

丈夫处理者:

public class Husband extends Handler {

    public Husband() {
        super(HUSBAND_LEVEL_REQUEST);
    }

    //丈夫自己的处理方法    
    @Override
    void response(IWomen women) {
        System.out.println("------妻子向成丈夫请求--------");
        System.out.println("请求 : " + women.getRequest());
        System.out.println("丈夫回复:同意");
    }

}

儿子处理者:

public class Son extends Handler {

    public Son() {
        super(SON_LEVEL_REQUEST);
    }

    //儿子自己的处理方法
    @Override
    void response(IWomen women) {
        System.out.println("------母亲向儿子请求--------");
        System.out.println("请求 : " + women.getRequest());
        System.out.println("儿子回复:同意");
    }

}

4、测试

public class Test {

    public static void main(String[] args) {

        //生成5个IWomen的对象，角色随机
        Random ran = new Random();
        List<IWomen> list = new ArrayList<IWomen>();
        for (int i = 0; i < 5; i++) {
            list.add(new Women(ran.nextInt(4), "行街购物"));
        }

        //链关系设置
        Handler father = new Father();
        Handler husband = new Husband();
        Handler son = new Son();
        father.setNext(husband);
        husband.setNext(son);

        //处理，由父亲开始
        for (IWomen w : list) {
            father.handleMessage(w);
        }

    }
}

例子二

Java WEB里面的Filter技术中用到的也是责任链的设计模式，多个Filter组成一个Filter链。下面例子模拟下Filter，实际中的Filter实现是很复杂的，例如怎么匹配URL拦截，Filter的执行顺序等等这些的实现，非常复杂，这里只是做一简单的实现。

例子

1、简单定义下Request和Response

Request:

public class Request {
    private String request;

    public String getRequest() {
        return request;
    }

    public void setRequest(String request) {
        this.request = request;
    }

}

Response:

public class Response {
    private String response;

    public String getResponse() {
        return response;
    }

    public void setResponse(String response) {
        this.response = response;
    }
}

2、定义一个Filter接口，派生出一个FilterChain主要Filter，另外还派生出两个自定义Filter.

Filter接口:

public interface Filter {
    void doFilter(Request request, Response response, FilterChain chain);
}

FilterChain主Filter:

public class FilterChain implements Filter {

    private List<Filter> filters = new ArrayList<Filter>();
    private int index = 0;

    public FilterChain add(Filter filter) {
        filters.add(filter);
        return this;
    }

    @Override
    public void doFilter(Request request, Response response, FilterChain chain) {
        if (index == filters.size()) {
            return;
        }
        Filter f=filters.get(index);
        index++;
        f.doFilter(request, response, chain);
    }

}

FilterChain所做的事只有两件，一个是拦截请求，第二个是添加Filter链。链的转发在doFilter方法之中。

自定义HTMLFilter用于转义Request里面的特殊字符:

public class HTMLFilter implements Filter {

    @Override
    public void doFilter(Request request, Response response, FilterChain chain) {
        // System.out.println("HTMLFilter");
        request.setRequest(request.getRequest().replace("<", "[").replace(">", "]"));
        chain.doFilter(request, response, chain);
        response.setResponse(response.getResponse() + "HTMLFilter");
}

}

自定义SensitiveFilter用于拦截Request里面的敏感词

public class SensitiveFilter implements Filter {

    @Override
    public void doFilter(Request request, Response response, FilterChain chain) {
        // System.out.println("SensitiveFilter");
        request.setRequest(request.getRequest().replace("MMP", "***").replace("敏感", "**"));
        chain.doFilter(request, response, chain);
        response.setResponse(response.getResponse() + "SensitiveFilter");
    }

}

3、测试

public class TestFilter {
    public static void main(String[] args) {
        String msg = "Test，<script></script>，MMP，敏感";
        FilterChain fc = new FilterChain();
        fc.add(new HTMLFilter()).add(new SensitiveFilter());
        Request request = new Request();
        request.setRequest(msg);
        Response response = new Response();
        response.setResponse("response");
        fc.doFilter(request, response, fc);
        System.out.println(request.getRequest());
        System.out.println(response.getResponse());

    }

}

总结

  • 责任链设计模式使多个对象都有机会处理请求，从而避免了请求的发送者和接受者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有对象处理它为止。

  • 责任链模式的重点是在“链”上，由一条链去处理相似的请求在链中决定谁来处理这个请求，并返回相应的结果。

  • 在实际应用中，一般会有一个封装类对责任模式进行封装，也就是替代Client类，直接返回链中的第一个处理者，具体链的设置不需要高层次模块关系，这样，更简化了高层次模块的调用，减少模块间的耦合，提高系统的灵活性。

责任链模式的优点

责任链模式非常显著的优点是将请求和处理分开。请求者可以不用知道是谁处理的，处
理者可以不用知道请求的全貌（例如在J2EE项目开发中，可以剥离出无状态Bean由责任链处
理），两者解耦，提高系统的灵活性。

责任链模式的缺点

责任链有两个非常显著的缺点：一是性能问题，每个请求都是从链头遍历到链尾，特别
是在链比较长的时候，性能是一个非常大的问题。二是调试不很方便，特别是链条比较长，
环节比较多的时候，由于采用了类似递归的方式，调试的时候逻辑可能比较复杂。

责任链模式的注意事项

链中节点数量需要控制，避免出现超长链的情况，一般的做法是在Handler中设置一个
最大节点数量，在setNext方法中判断是否已经是超过其阈值，超过则不允许该链建立，避免
无意识地破坏系统性能。
