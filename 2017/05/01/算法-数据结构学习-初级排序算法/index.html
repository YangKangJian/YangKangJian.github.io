算法&数据结构学习-初级排序算法

发表于 2017-05-01

看一眼书架上的布满灰尘的橙汁色书《算法》，回想起当初在大学的时候那种对数据结构和算法的那种疯狂，认为没有什么结构能比得上一棵二叉树还美丽，看到什么生活例子都会映射到数据结构中来，排个队也会想将队列扭转成栈。如今出来工作了，多数和业务和框架这些打交道了，当年在学校学习这本书被尘封在书架上，有些算法没用到已经遗忘得差不多了。觉得，是时候再次学习了，顺便写写总结。《算法》这本书虽然这本书仅仅是入门级的，但非常推荐去学，作为一个日常的应用层码农，学习这本书，足够应对工作中出来的算法和数据结构问题了，当然，不是设计算法这些工作了。

今天来学习下两个基础的简单排序算法，选择排序和插入排序。

初级排序算法

准备工作

《算法》这本书用Java语言实现的，通篇中让我觉得最好的地方就是规范，先设计出接口，然后面向接口进行编程。书中排序算法抽象出了这么一个排序的类:

public abstract class Sort<T> {
    public abstract void sort(Comparable<T>[] a);

    //比较对象大小，判断v是否小于于w
    @SuppressWarnings("unchecked")
    public boolean less(Comparable<T> v, Comparable<T> w) {
        return v.compareTo((T) w) < 0;
    }

    //交换
    public void exch(Comparable<T>[] a, int i, int j) {
        Comparable<T> t = a[i];
        a[i] = a[j];
        a[j] = t;
    }

    //打印
    public void show(Comparable<T>[] a) {
        for (int i = 0; i < a.length; i++) {
            System.out.print(a[i].toString() + ",");
        }
    }

    //是否已经排序了
    public boolean isSorted(Comparable<T>[] a) {
        for (int i = 1; i < a.length; i++) {
            if (less(a[i], a[i - 1])) {
                return false;
            }
        }
        return true;
    }

}

在Java中Comparable是一个比较器接口，一个类实现了Comparable接口可以通过compareTo方法的实现对象的比较，compareTo方法也被称为自然比较方法。如果一个Collection的对象想要通过Collections的sort方法实现排序的，那么这个对象必须实现Comparable接口。compareTo方法的返回值是int，有三种情况：

  • 比较者大于被比较者，那么返回1

  • 比较者等于被比较者，那么返回0

  • 比较者小于被比较者，那么返回-1

Java中的基本数据类型的包装类和String这些都实现了Comparable这个接口，所以，上面的Sort这个抽象类适合于这些实现Comparable接口的对象进行排序。

选择排序

选择排序（Selection sort）是一种简单直观的排序算法。它的工作原理如下。首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置，然后，再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。以此类推，直到所有元素均排序完毕。

例如，将数组元素进行顺序排列，最简单的选择排序算法思路是这样的:

1、首先找出数组元素中最小的元素，和数组的第一个元素交换(如果第一个元素是最小的就和自己交换)

2、然后再在剩下的元素中找出最小的元素，将它与第二个元素进行交换，以此类推，直到最后一个元素为止。

所以这种方法就是在剩余的元素之中不断选择最小的元素，所以叫选择排序。

实现

public class SelectSort extends Sort<Integer> {

    @Override
    public void sort(Comparable<Integer>[] a) {
        int N = a.length;
        for (int i = 0; i < N; i++) {
            int min = i;
            for (int j = i + 1; j < N; j++) {
                if (less(a[j], a[min])) {
                    min = j;
                }
                exch(a, i, min);
            }
        }
    }

}

测试

public class Main {
    public static void main(String[] args) {
        Integer[] a = new Integer[] { 1, 5, 4, 7, 8, 9, 2, 9, 4, 5, 3, 6, 12, 1, };
        Sort<Integer> selectSort = new SelectSort();
        selectSort.sort(a);
        selectSort.show(a);

    }
}
//out:1,2,3,4,4,5,5,7,8,9,9,

在实现的代码之中，外循环遍历整个数组，而内循环就是从i+1位置开始遍历寻找出最小的元素下标，然后将这个最小的元素和a[i]进行交换。交换元素逻辑在内循环外，因此每次的外循环都会发生一次的交换，因此交换的次数是N。算法的时间效率取决于交换次数。

  • 选择排序的主要优点与数据移动有关。
  • 对于长度为N的数组，选择排序需要大约N^2/2次比较和N次的交换。
  • 选择排序有两个简单鲜明的特点:
    • 1:运行时间和输入无关。为了找出最小的元素而去扫描整个数据，如果一个数组接近有序的，这几乎是浪费时间。
    • 2:数据移动最小。每次交换都改变两个元素的值，交换次数和数组长度一致，呈线性关系。
  • 在所有的完全依靠交换去移动元素的排序方法中，选择排序属于非常好的一种。

插入排序

插入排序（Insertion Sort）是一种简单直观的排序算法。它的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。插入排序在实现上，通常采用in-place排序（即只需用到O(1)的额外空间的排序），因而在从后向前扫描过程中，需要反复把已排序元素逐步向后挪位，为最新元素提供插入空间。

一般来说，插入排序都采用in-place在数组上实现。具体算法描述如下：

1、从第一个元素开始，该元素可以认为已经被排序

2、取出下一个元素，在已经排序的元素序列中从后向前扫描

3、如果该元素（已排序）大于新元素，将该元素移到下一位置

4、重复步骤3，直到找到已排序的元素小于或者等于新元素的位置

5、将新元素插入到该位置后

6、重复步骤2~5

实现

public class Insertion extends Sort<Integer> {

    @Override
    public void sort(Comparable<Integer>[] a) {
        int N = a.length;
        for (int i = 0; i < N; i++) {
            for (int j = i; j > 0 && less(a[j], a[j - 1]); j--) {
                exch(a, j, j - 1);
            }
        }
    }
}

测试

public class Main {
    public static void main(String[] args) {
        Integer[] a = new Integer[] { 1, 5, 4, 7, 8, 9, 2, 9, 4, 5, 3, 6, 12, 1, };
        Sort<Integer> insertSort = new Insertion();
        insertSort.sort(a);
        insertSort.show(a);

    }
}
